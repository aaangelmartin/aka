## ¿Por qué `aka`?

Tu `.zshrc` (o `.bashrc`, o `config.fish`) es un cementerio de aliases
que ya nadie recuerda haber añadido, sin búsqueda, sin categorías, sin
vista previa. Editarlo a mano es frágil — una comilla mal puesta y tu
shell no carga.

`aka` (a.k.a. — *also known as*) resuelve eso:

- **Un único lugar para todos tus aliases**, respaldado por un archivo
  JSON.
- **TUI + CLI** para añadir, editar, borrar, buscar, etiquetar.
- **Multi-shell**: genera `aliases.zsh`, `aliases.bash` y
  `aliases.fish` desde la misma fuente.
- **Seguro por diseño**: tu rc se modifica una sola vez (una línea
  `source …` entre marcas `# >>> aka >>>`). `aka uninstall` limpia.
- **Bilingüe** (inglés · español).

## Instalación

### Homebrew (macOS / Linux)

```sh
brew install aaangelmartin/tap/aka
```

### Go

```sh
go install github.com/aaangelmartin/aka/cmd/aka@latest
```

### Manual

Descarga un binario desde la
[última release](https://github.com/aaangelmartin/aka/releases/latest),
extráelo y coloca `aka` en tu `$PATH`.

## Empezar rápido

```sh
aka install zsh                # añade `source ~/.config/aka/aliases.zsh` a ~/.zshrc
aka import --from-rc ~/.zshrc  # migra los aliases existentes
aka add cc claude              # añade el alias: cc='claude'
aka ls                         # lista aliases
aka                            # lanza la TUI
```

## Tour rápido

```console
$ aka --lang es add cc claude
añadido cc → claude

$ aka --lang es add -t git gs 'git status'
añadido gs → git status

$ aka --lang es ls
NAME  SHELLS         TAGS  COMMAND
cc    zsh,bash,fish        claude
gs    zsh,bash,fish  git   git status

$ cat ~/.config/aka/aliases.zsh
# Generated by aka — do not edit by hand.
# Edit with: aka edit <name>   |   aka   (TUI)
# Source of truth: ~/.local/share/aka/aliases.json

alias cc='claude'
alias gs='git status'
```

Ejecuta `aka` sin argumentos para abrir la TUI: flechas para
moverte, `/` filtrar, `a` añadir, `e` editar, `d` borrar, `enter`
copia el comando al portapapeles, `?` ayuda.

## Configuración

La configuración vive en `~/.config/aka/config.toml` (o
`$XDG_CONFIG_HOME/aka/`). Lee o escribe con `aka config <clave> <valor>`:

```sh
aka config language es              # fija el idioma a español
aka config theme dracula            # default | dracula | nord | gruvbox
aka config default_shells zsh,bash  # limita los archivos generados a estos shells
```

Las rutas se pueden forzar con `$AKA_CONFIG`, `$AKA_DATA` y
`$AKA_OUTDIR` — útil para tests o setups portátiles.
